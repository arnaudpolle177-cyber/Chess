--------------------------------------------------------------------
0. OBJECTIF
--------------------------------------------------------------------
Ne plus CHOISIR un commentaire (1 thème -> 1 gabarit).
Mais CONSTRUIRE un raisonnement, comme un vrai entraîneur :
observer plusieurs éléments de la position, sélectionner les 2-3 idées
les plus importantes, et les tisser en UNE réflexion cohérente.

Le lecteur ne doit jamais sentir que des commentaires ont été collés.
Il doit lire une seule pensée fluide de 2 à 4 phrases.

CONSIGNE FINALE (la plus importante) :
Si un thème secondaire n'améliore pas réellement la compréhension,
NE PAS l'utiliser. Mieux vaut 1 excellente idée qu'un commentaire
artificiellement enrichi.

--------------------------------------------------------------------
1. DÉCISION D'ARCHITECTURE VERROUILLÉE
--------------------------------------------------------------------
Assemblage retenu : FRAGMENTS + CONNECTEURS RELATIONNELS.
  - PAS de LLM (refusé).
  - PAS de templates par paire de thèmes (option écartée : explosion
    combinatoire, ~13 thèmes => trop de paires x 3 profils, ingérable,
    et contraire à "modulaire, pas un bloc de conditions").

Principe :
  - Chaque thème/brique fournit des FRAGMENTS courts, pas un texte fini :
        { observation, cause, plan }   (x3 voix : popular/creative/classical)
  - Le narrateur assemble avec un CONNECTEUR choisi selon la RELATION
    sémantique entre le thème principal et le secondaire.
  - Le PLAN vient TOUJOURS du thème principal (garde le paragraphe
    centré sur une seule idée directrice).

Pourquoi ça marche : la table de relations est petite et éparse (des
types de connecteurs, pas des phrases entières). Ajouter un thème =
ajouter ses fragments, rien d'autre à toucher (coût linéaire).

3. LES 4 PIÈCES À CONCEVOIR
--------------------------------------------------------------------

3.1 SCORING GLOBAL  (theme_scoring.py)
    score = poids_tier + intensité_normalisée(0..99)
    Tiers (d'après la priorisation demandée) :
      Très forte  = 1000 : BLUNDER, TACTICAL, ATTACK, DEFENSE
      Moyenne     =  500 : STRATEGIC_ADVANTAGE, ENDGAME, OPENING, MISSED_OPPORTUNITY
      Enrichisst. =  100 : PAWN_STRUCTURE, PIECE_ACTIVITY_GAP, MATERIAL_IMBALANCE,
                           KING_SAFETY_WARNING, INITIATIVE_SHIFT, SIMPLIFICATION
    L'intensité (0..99) départage AU SEIN d'un tier (ex: swing_cp pour
    BLUNDER, |eval_cp| pour STRATEGIC, activity_ratio pour ACTIVITY...).
    Principal = score max.

3.2 SÉLECTION + ANTI-REDONDANCE
    - Chaque thème reçoit une FAMILLE : ex. structure / roi / activité /
      dynamique / matériel / phase / tactique.
    - Deux thèmes de la MÊME famille ne coexistent pas -> garder le mieux scoré
      (ex: PAWN_STRUCTURE et STRATEGIC_ADVANTAGE racontent souvent la même
      idée "structure meilleure" -> un seul).
    - Prendre jusqu'à 2 secondaires qui :
        (a) dépassent un SEUIL plancher,
        (b) sont d'une famille DIFFÉRENTE du principal,
        (c) ont une relation UTILE avec le principal.
    - Si aucun secondaire n'apporte de dimension neuve -> 0 secondaire.
      (Consigne finale : un excellent commentaire à 1 idée > un enrichi bancal.)

3.3 TABLE DE RELATIONS  (petite)
    relation(principal, secondaire) -> type_de_connecteur
    Types suffisants : cause / conséquence / moyen / contraste / renforcement.
    Défaut si paire non listée : connecteur neutre ("Par ailleurs...").
    Exemples :
      structure  -> activité   = cause->effet      "...grâce à..., ce qui permet à tes pièces de..."
      roi_faible + activité     = problème+moyen    "...est l'élément clé. Tes pièces tiennent déjà
                                                     les lignes, il est donc préférable de..."
      avantage + simplification = constat+plan      "...; pas besoin de forcer, cherche plutôt à..."

3.4 LE WEAVER  (narration_weaver.py)
    Assemble : observation(principal)
             + connecteur(relation)
             + observation/cause(secondaire)
             + plan(principal)
    Décliné par voix de profil (popular / creative / classical).
    Le plan vient toujours du PRINCIPAL.

5. POINTS DE VIGILANCE
--------------------------------------------------------------------
- PARTAGE DE THÈME : aujourd'hui le thème est calculé 1 fois/position et
  réutilisé par les 3 profils (cache). En v2 : la SÉLECTION est
  indépendante du profil -> se cache 1 fois ; seul le WEAVING varie.
  Cohérent avec l'archi existante.
- MATERIAL_IMBALANCE / SIMPLIFICATION : aujourd'hui ce sont des CHAMPS de
  STRATEGIC_ADVANTAGE, pas des thèmes. Décision penchée : en faire des
  BRIQUES à part entière (scoring uniforme, plus propre). À confirmer.
- CONTRAINTES ADN À NE JAMAIS CASSER :
    * Rien d'inventé : chaque fragment s'appuie sur un champ RÉEL du
      ThemeResult (swing_cp, eval_cp, king_square, pawn_weakness_*,
      material_imbalance_kind, simplification_advice, activity_ratio,
      king_safety_warning_*, opponent_better_move_san, passed_pawn_square).
    * Hors-ligne / gratuit par défaut.
    * CLAUDE.md local : séparer overlay (flèches) et coach (explications) ;
      préférer extraire plus d'info d'une analyse existante plutôt que
      multiplier les appels moteur.
